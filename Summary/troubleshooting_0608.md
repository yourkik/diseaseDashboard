# Disease Dashboard - Troubleshooting Document (지도 및 AI 고도화 - 06.08)

본 문서는 질병관리청(KDCA) 공공데이터 연동, 프론트엔드 Azure 지도 시각화 및 AI 뉴스 분석 과정에서 발생한 주요 에러와 그 해결 과정을 정리한 트러블슈팅 로그입니다.

---

## 1. 지도 모드 토글 시 버블이 증발하는 현상 (React Mount Race Condition)

- **증상:** 기존 정적 지도(SVG)에서 최신 버전 지도(Azure Maps)로 토글 스위치를 누르면 지도는 정상적으로 뜨나, 확진자 버블이나 애니메이션 선이 나타나지 않는 현상 발생.
- **원인:** 토글 시 컴포넌트가 새롭게 마운트되며 두 가지 비동기 작업이 시작됨. 1) 백엔드 API에서 확진자 통계 가져오기(`fetch`) 2) 무거운 Azure Maps WebGL 엔진 로딩. 로컬 서버의 API 응답이 지도 엔진 로딩보다 훨씬 빨리 끝나버려, 지도가 준비되지도 않았는데 데이터를 그리라는 명령이 내려짐. 그 결과 지도는 명령을 무시했고 이후 재요청이 일어나지 않음.
- **해결:** `mapReady`라는 상태(State) 값을 도입. 지도 엔진이 완벽히 부팅되어 `map.events.add('ready')`가 호출될 때 `setMapReady(true)`를 트리거하고, `useEffect` 의존성 배열에 `mapReady`를 추가하여 지도가 준비된 후 안전하게 데이터를 렌더링하도록 동기화 파이프라인을 구축함.

## 2. 텍스트 라벨 추가 시 파란색 기본 핀이 겹쳐 보이는 버그

- **증상:** 버블 위에 확진자 수를 상시 표기하기 위해 `SymbolLayer`를 추가하자, 의도치 않게 Azure Maps의 기본 파란색 마커(Map Pin) 아이콘이 렌더링되어 커다란 붉은 버블 위에 작은 파란 핀이 박혀 있는 듯한 어색한 UI가 됨.
- **원인:** Azure Maps의 `SymbolLayer`는 텍스트만 입력하더라도 디폴트로 아이콘(마커)을 함께 렌더링하려는 성질이 있음.
- **해결:** `SymbolLayer` 생성 옵션에 `iconOptions: { image: 'none' }` 속성을 명시적으로 선언하여, 기본 핀 아이콘 렌더링을 끄고 오직 텍스트만 깔끔하게 노출되도록 패치함.

## 3. Azure OpenAI 의료/안전 필터에 의한 답변 거부(차단) 문제

- **증상:** 백일해, 유행성이하선염 등 특정 질병에 대한 뉴스 요약을 요청할 때, AI가 정상적인 분석 리포트를 주지 않고 `"I'm sorry, but I cannot assist with that request."`라는 메시지만 반환하는 현상 발생.
- **원인:** Azure OpenAI의 Responsible AI (안전 필터) 정책이 질병의 증상, 사망자 수, 감염 확산 등의 뉴스 키워드를 '위험한 의료 상담(Medical Advice)' 또는 '유해 콘텐츠'로 오인하여 AI의 응답을 강제로 차단(Filtered)해 버림.
- **해결:** 백엔드의 시스템 프롬프트(Prompt Engineering)를 대폭 보강하여, 해당 봇의 역할이 **"의료 상담이나 자가 진단 목적이 아닌, 공공 보건 대시보드를 위한 객관적인 통계 및 전염병학적 분석(Epidemiological analysis)임"**을 AI에게 강하게 인지시킴. 이를 통해 안전 필터를 우회하여 정상적인 AI 종합 분석 리포트를 받아내는 데 성공함.
