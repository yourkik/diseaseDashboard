# Disease Dashboard - Troubleshooting Document (API 및 지도 렌더링)

본 문서는 질병관리청(KDCA) 공공데이터 API 연동 및 프론트엔드 지도 시각화 과정에서 발생한 주요 에러와 그 해결 과정을 정리한 트러블슈팅 로그입니다.

---

## 1. FastAPI 422 Unprocessable Entity 에러

- **증상:** 프론트엔드에서 백엔드 API(`/api/stats/map/spread`)를 호출할 때 데이터가 반환되지 않고 HTTP 422 에러가 발생함.
- **원인:** 백엔드의 FastAPI 라우터는 쿼리 파라미터로 `disease_name`을 기대하고 있었으나, 프론트엔드와 테스트 스크립트에서는 `disease`라는 이름으로 파라미터를 넘겨주어 파라미터 유효성 검사를 통과하지 못함.
- **해결:** 백엔드 `disease_stats.py`의 라우터 파라미터를 `disease`로 변경하고, 내부 함수 호출 시에도 통일된 키워드 인자를 사용하도록 수정하여 매핑 불일치를 해소함.

## 2. 긴 로딩 시간 및 타임아웃 문제 (캐싱 부재)

- **증상:** 지도를 렌더링하기 위해 API를 호출하면 30~40초의 긴 지연 시간이 발생하거나 브라우저에서 타임아웃이 발생함.
- **원인:** 백엔드에서 전국 시계열 데이터를 모으기 위해 17개 시도 코드를 순회하며 순차적으로 동기(Sync) 외부 HTTP 요청(총 30~40회)을 보내기 때문에 병목이 발생함.
- **해결:** `kdca_service.py`에 **로컬 파일 기반의 영구 캐싱 시스템**(`kdca_cache.json`)을 도입. 한 번 수집된 데이터는 24시간 동안 캐싱 파일에서 읽어오도록 수정하여, 최초 호출 이후에는 0.1초 만에 즉각적으로 지도를 렌더링하도록 성능을 극적으로 개선함.

## 3. 콤마(,) 포함 문자열로 인한 500 서버 에러

- **증상:** 수두와 같이 확진자 수가 많은 질병을 조회할 때 백엔드 서버가 500 에러를 뱉으며 뻗음 (`invalid literal for int() with base 10: '26,964'`).
- **원인:** 질병관리청 API가 확진자 수가 1,000명이 넘어갈 경우 데이터에 쉼표(,)를 포함한 문자열(`"26,964"`)로 반환함. 이를 파이썬의 `int()`로 강제 형변환하려다 런타임 에러가 발생함.
- **해결:** `disease_stats.py` 내부에 `safe_int()`와 `safe_float()` 헬퍼 함수를 구현하여, 문자열에 포함된 쉼표(,)를 사전에 `.replace(",", "")`로 제거한 후 안전하게 형변환하도록 패치함.

## 4. 모든 질병의 확산 지도가 동일하게 나타나는 버그 (KDCA 지역 코드 스펙 문제)

- **증상:** 수두, 백일해, 유행성이하선염 등 모든 질병의 지도가 "부산 -> 대구, 인천, 광주"로 동일한 화살표를 그리고, 서울을 제외한 모든 지역의 확진자 수가 동일한 값(전국 총합)으로 표시됨.
- **원인:** 백엔드에서 통계청 표준 행정구역 코드(`11`: 서울, `21`: 부산 등)를 사용하여 질병관리청 API를 호출했으나, **질병관리청 API는 이 표준 코드를 인식하지 못함.** 더 큰 문제는 알 수 없는 코드가 들어왔을 때 에러를 뱉지 않고 **"전국 총합(00)" 데이터를 무시하고 던져주는 비정상적인 스펙**을 가지고 있었음. 이로 인해 모든 지역이 전국 총합 데이터를 부여받아 정렬 알고리즘이 항상 동일한 순위를 반환함.
- **해결:** 질병관리청 서버를 상대로 1~50까지의 코드를 찔러보는 **리버스 엔지니어링(스캐너) 스크립트**를 작성하여 백엔드에서 실행함. 그 결과 질병관리청은 독자적인 `01`~`17` 순번 코드를 사용한다는 것을 밝혀내고(`01`: 서울, `02`: 부산 ... `17`: 세종), `SIDO_MAPPING` 딕셔너리를 해당 정답 코드로 전면 교체하여 해결함.

## 5. 지도의 화살표와 툴팁 수치의 불일치 (지역명 정규화 버그)

- **증상:** 마커와 화살표는 "경남"을 가리키고 있는데, 정작 해당 지역의 다각형에 마우스를 올리면 툴팁에 "경상남도 0명"이라고 뜨고 지도 색상도 검게(0명) 칠해져 있음.
- **원인:** 지도의 다각형을 그리는 `korea-topo.json` 데이터에는 지역명이 `경상남도`로 되어 있으나, 백엔드가 주는 API 데이터는 `경남`으로 되어 있음. 이를 매칭하기 위해 사용된 `substring(0, 2)` 로직이 `경상남도`를 `경상`으로 잘라버려, API 데이터의 `경남` 키를 찾지 못하고 `undefined`(0명)로 처리됨. 반면 화살표 연산 로직은 API 데이터 키(`경남`)를 직접 사용했기 때문에 화살표는 정상적으로 그려지는 엇박자가 발생함.
- **해결:** `InfectionMap.jsx`의 `normalizeRegion` 함수를 수정하여 `name.includes("경상남도") return "경남"`과 같이 풀네임(Full name)과 줄임말 간의 **1:1 하드코딩 매칭 로직**을 견고하게 추가하여, 툴팁 수치와 지도 색상이 정상적으로 렌더링되도록 수정함.
