# 감염병 대시보드 프로젝트 종합 기획서 (Plan.md)

본 문서는 시민들의 자가 방역을 돕고 보건소의 선제적 대응을 지원하기 위한 **'감염병 대시보드'**의 목적, 기술 스택, 시스템 아키텍처 및 개발 방향성을 총망라한 최종 기획서입니다.

---

## 1. 프로젝트 개요 및 목표

- **대국민 서비스 (시민 레벨):** 비전문가도 쉽게 이해할 수 있는 직관적인 위험도 정보 및 AI 기반 감염병 뉴스 요약, 자가 방역 지침 제공.
- **B2G 서비스 (공공/정부 레벨):** 특정 지역 내 뉴스 언급량 급증 또는 확진자 수 임계치 돌파 시, 보건소 담당자에게 알림을 발송하여 신속한 역학조사 및 대응 지원.

---

## 2. 시스템 아키텍처 및 기술 스택

본 프로젝트는 확장성과 유지보수를 고려하여 **하이브리드(Docker 로컬 개발 후 Azure 클라우드 배포)** 방식을 채택합니다.

### 2.1 프론트엔드 (대시보드 및 관리자 UI)
- **프레임워크:** **Next.js (React App Router)**
- **디자인/UI:** Premium Dark Mode, Glassmorphism, Google Fonts(`Inter`, `Outfit`)를 적용한 미려한 반응형 웹 디자인.
- **주요 페이지:**
  - `Public Dashboard (/):` 모의 확진자 통계, 현재 위험도 배지, RAG 기반 뉴스 브리핑, 추후 연동될 **지도 시각화(Heatmap) 영역** 포함.
  - `Admin Setup (/admin):` 보건소 담당자가 비개발자라도 직접 알림 발송 조건(확진자 수 임계치, 위험 키워드, 수신 이메일)을 설정할 수 있는 제어 페이지.

### 2.2 데이터 수집 (Data Collection Pipeline)
크롤링을 배제하고 공식 API를 활용하여 신뢰도와 안정성을 확보합니다.
- **정형 데이터:** 질병관리청 공공데이터포털 API (확진자 통계, 백신 접종률 등)
- **비정형 데이터 (통합 수집 구조):** 
  - **Bing News Search API:** 우선적으로 고품질의 국내외 뉴스를 수집합니다.
  - **GDELT Project API (병행/Fallback):** 전 세계 이벤트 데이터를 수집하는 완전 무료 오픈소스 API입니다. Bing API 키가 없거나 제한에 걸렸을 때의 대체재(Fallback)이자, 추가적인 글로벌 트렌드 교차 검증을 위해 병행 호출합니다.
- **개발 환경:** 초기 개발은 **Python 또는 Node.js와 Docker**를 활용하여 로컬에서 빠르게 구축 및 테스트 진행.

### 2.3 AI 전처리 및 RAG 파이프라인
수집된 방대한 비정형 뉴스 데이터에서 유의미한 정보를 추출하기 위해 RAG(Retrieval-Augmented Generation) 구조를 적용합니다.
- **검색 및 색인 (Retrieval):** **Azure AI Search**를 통해 뉴스 문서들을 벡터화(Vectorization)하여 저장하고 빠른 유사도 검색 수행.
- **추론 및 생성 (Generation):** **Azure OpenAI (GPT 모델)**를 활용하여, 검색된 뉴스를 바탕으로 시민용 위험도 분석 및 행동 지침 요약 텍스트 생성. 해당 결과를 대시보드에 직접 노출.

### 2.4 데이터 저장소 (Database)
데이터의 성격에 맞춰 저장소를 분리하여 조회 성능을 높이고 비용을 최적화합니다.
- **정형 데이터 (시계열 통계):** **PostgreSQL** (로컬 Docker 컨테이너 활용 후 클라우드 마이그레이션)
- **원문 백업:** Azure Blob Storage (데이터 레이크 목적)
- **벡터 데이터:** Azure AI Search 자체 저장소

---

## 3. 핵심 기능 동작 흐름 (Workflow)

1. **데이터 파이프라인 가동:** 스케줄러(초기 로컬 Cron / 향후 클라우드 환경)가 질병관리청과 Bing API에서 데이터를 수집해 DB에 저장.
2. **AI(RAG) 분석 가동:** 새로 들어온 뉴스들을 Azure OpenAI가 분석하여 위험도를 분류하고 알림 조건(Admin 설정값)과 대조.
3. **조건 충족 시 알림:** Admin에서 설정한 조건(예: '독감' 키워드 하루 50회 이상)에 도달하면 보건소 담당자에게 이메일/문자 발송.
4. **시민 대시보드 노출:** 시민들은 Next.js 웹에 접속하여 최신화된 확진자 통계, 지도 시각화, AI 뉴스 요약 리포트를 열람.

## 4. 향후 개발 마일스톤 (Milestones)

> [!TIP]

> - 요구사항 분석, 프레임워크 선정, 데이터 소스 확정 및 UI 프로토타이핑(로컬 Next.js 세팅 완료).
> 
> **Phase 2: 로컬 백엔드 및 데이터 파이프라인 구축 (Next Step)**
> - Docker 기반 로컬 환경에 PostgreSQL 세팅.
> - Python/Node.js로 공공데이터 API 및 Bing News API 실제 호출 스크립트 작성.
> 
> **Phase 3: AI / RAG 모듈 연동**
> - Azure OpenAI 및 Azure AI Search 리소스 개설 후 API 연동 테스트.
> 
> **Phase 4: 프론트엔드 연동 및 시각화**
> - 대시보드 지도(Azure Maps 등) 연동 및 수집된 백엔드 데이터를 Next.js 화면에 뿌려주는 API 개발.
> 
> **Phase 5: 클라우드 배포 (Deployment)**
> - 완성된 Docker 컨테이너(Front/Back/DB)를 Azure Container Apps 또는 App Service로 배포하여 실제 서비스화.
